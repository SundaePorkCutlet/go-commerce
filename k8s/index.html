go-commerce Kubernetes Study Map

Docker Compose 기반 MSA를 kind Kubernetes 환경으로 옮기며, 배포 단위와 서비스 간 통신을 학습하기 위한 요약 문서입니다.

Deployment 서비스 Pod를 원하는 개수만큼 유지하고 롤아웃을 관리합니다.
Service 바뀌는 Pod 앞에 고정 DNS와 포트를 제공합니다.
ConfigMap DB host, Kafka broker 같은 일반 설정을 이미지 밖에서 주입합니다.
Secret DB password, JWT secret, Xendit key 같은 민감 정보를 분리합니다.

Deployment Roadmap

Phase 1. ORDERFC + 최소 의존성 kind cluster, postgres-order, redis, ORDERFC, health check
Phase 2. Dependencies PostgreSQL, Redis, Kafka를 Service DNS로 연결
Phase 3. Saga ORDERFC → PRODUCTFC → PAYMENTFC 이벤트 흐름 검증

Compose to Kubernetes

Docker Compose Kubernetes 학습 포인트
service Deployment + Service 실행 단위와 접근 주소를 분리합니다.
environment ConfigMap / Secret 설정과 민감 정보를 이미지 밖에서 관리합니다.
depends_on readinessProbe / retry Kubernetes는 시작 순서보다 준비 상태와 복구를 중시합니다.
ports Service / Ingress 클러스터 내부 접근과 외부 노출을 분리합니다.

First Commands

kind create cluster --name go-commerce
docker build -t go-commerce-orderfc:local ./ORDERFC
kind load docker-image go-commerce-orderfc:local --name go-commerce

kubectl apply -f k8s/manifests/namespace.yaml
kubectl apply -f k8s/manifests/orderfc.yaml

kubectl get pods -n go-commerce
kubectl get svc -n go-commerce
kubectl logs deployment/orderfc -n go-commerce
