Partition Key 기반으로 동일 사용자 이벤트의 순서를 보장
DLQ 토픽으로 실패 이벤트 재처리 루트를 확보
Redis 멱등 키로 중복 소비·중복 반영 가능성 완화
결제 실패 시 stock.rollback으로 보상 처리 일관성 유지
※ 주문 저장과 이벤트 발행의 원자성은 추후 Outbox 패턴으로 강화 가능
